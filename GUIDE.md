## 1. 홈페이지 구조 개요

홈페이지는 총 **3개의 HTML 파일**로 구성됩니다.

| 파일 | 역할 | 접근 권한 |
|------|------|-----------|
| `index.html` | 공개 랜딩 페이지 | 누구나 접근 가능 |
| `dashboard.html` | 회원 전용 대시보드 | 회원 로그인 필요 |
| `admin.html` | 관리자 전용 패널 | 관리자 로그인 필요 |

```
홈페이지 주소 (예시)
https://YOUR_USERNAME.github.io/highend-volleyball/          ← 공개 페이지
https://YOUR_USERNAME.github.io/highend-volleyball/dashboard.html  ← 회원 로그인
https://YOUR_USERNAME.github.io/highend-volleyball/admin.html      ← 관리자
```

### 각 페이지 기능

**index.html (공개 페이지)**
- 아카데미 소개 및 코치 프로필
- 수업 일정 달력 (로그인 없이 확인 가능, Firebase 실시간 연동)
- 커리큘럼 안내
- 사진 갤러리
- 원데이클래스 신청 링크 (네이버)
- 인스타그램 / 전화 / 카카오톡 링크

**dashboard.html (회원 전용)**
- 내 수강권 현황 (잔여 횟수, 만료일, 진행 바)
- 수업 예약 (달력 → 날짜 선택 → 예약 버튼)
- 예약 취소 (전날 22:00 이전까지)
- 수강권 신청 (계좌이체 후 신청)
- 비밀번호 변경

**admin.html (관리자 전용)**
- 스케줄 달력 (수업 생성 / 취소 / 출석 체크)
- 토요일 수업 자동 생성 (한 달치)
- 회원 등록 / 수정 / 삭제
- 결제 요청 승인 / 반려 (자동 수강권 발급)
- 데이터 백업 (JSON / CSV 다운로드)
- Firebase 연결 상태 확인

## 3. GitHub 배포 방법

### 사전 준비

1. **GitHub 계정** 생성: https://github.com
2. **Git 설치** (이미 설치됨): https://git-scm.com
3. **GitHub 저장소** 생성

### Step 1 — GitHub에서 새 저장소 만들기

1. https://github.com → 로그인
2. 우측 상단 **`+`** → **New repository**
3. Repository name: `highend-volleyball` (원하는 이름)
4. **Public** 선택 (GitHub Pages 무료 사용을 위해 반드시 Public)
5. README 체크 없이 **Create repository** 클릭

### Step 2 — 로컬 폴더를 GitHub에 업로드

VS Code 터미널 또는 명령 프롬프트에서 아래 명령 실행:

```bash
# 홈페이지 폴더로 이동
cd "c:\Users\User\0. 강습 홈페이지"

# GitHub 저장소 연결 (YOUR_USERNAME을 본인 GitHub 아이디로 변경)
git remote add origin https://github.com/YOUR_USERNAME/highend-volleyball.git

# GitHub에 업로드
git push -u origin main
```

> 처음 push 시 GitHub 로그인 팝업이 나타납니다. GitHub 아이디/비밀번호 입력.

### Step 3 — GitHub Pages 활성화

1. GitHub 저장소 페이지 → **Settings** 탭
2. 좌측 메뉴 → **Pages**
3. Source 섹션: **Deploy from a branch** 선택
4. Branch: **main** 선택, 폴더: **/ (root)** 선택
5. **Save** 클릭
6. 약 2~3분 후 배포 완료

**배포 완료 주소:**
```
https://YOUR_USERNAME.github.io/highend-volleyball/
```

> Pages 탭에서 초록색으로 "Your site is live at..." 메시지가 나타나면 배포 완료.

## 5. Firebase 데이터베이스 설정

### ⚠️ 필수 설정 — 이 작업을 하지 않으면 모든 쓰기 작업이 실패합니다

1. https://console.firebase.google.com 접속
2. 프로젝트 **high-end-f4c77** 선택
3. 좌측 메뉴 → **빌드** → **Realtime Database**
4. **규칙(Rules)** 탭 클릭
5. 아래 내용으로 교체:

```json
{
  "rules": {
    ".read": true,
    ".write": true
  }
}
```

6. **게시(Publish)** 버튼 클릭

### 데이터베이스 구조 (자동 생성됨)

```
Firebase Realtime Database
├── members/          ← 회원 정보
│   └── {id}/
│       ├── id
│       ├── name          (이름)
│       ├── phone         (전화번호, 로그인 아이디)
│       ├── password      (비밀번호)
│       ├── birthdate     (생년월일)
│       ├── classType     (group / private)
│       ├── passType      (4 / 8 / 12 / custom)
│       ├── regularSessions   (그룹 잔여 횟수)
│       ├── privateSessions   (프라이빗 잔여 횟수)
│       ├── expiryDate    (만료일)
│       └── isAdmin       (true / false)
│
├── sessions/         ← 수업 일정
│   └── {id}/
│       ├── date          (YYYY-MM-DD)
│       ├── time          (HH:MM)
│       ├── title         (수업 제목)
│       ├── sessionType   (group / private)
│       ├── maxCapacity   (최대 인원, 기본 16)
│       └── status        (없음=정상 / cancelled=취소)
│
├── reservations/     ← 예약 내역
│   └── {id}/
│       ├── memberId      (회원 ID)
│       ├── sessionId     (수업 ID)
│       ├── memberName    (회원 이름)
│       ├── memberPhone   (회원 전화번호)
│       ├── date, time, title
│       ├── status        (confirmed / cancelled)
│       └── createdAt
│
├── attendance/       ← 출석 기록
│   └── {sessionId}_{memberId}: {timestamp}
│
└── paymentRequests/  ← 수강권 신청
    └── {id}/
        ├── memberId, memberName, phone
        ├── classType, passType, note
        ├── status        (pending / confirmed / rejected)
        └── createdAt
```

## 6. 관리자 패널 사용법

접속: `/admin.html` → 아이디 `admin` / 비밀번호 `s1102!`

### 탭 1: 스케줄 (수업 캘린더)

**달력 사용법**
- 달력에서 날짜 클릭 → 해당 날짜의 수업 목록 표시
- 파란 점: 정상 수업 있음
- 회색 점: 취소된 수업

**토요일 자동생성 버튼**
- 오늘부터 30일 이내 모든 토요일에 3타임 자동 생성
- 12:00 / 13:30 / 16:30 각 16명 정원
- 이미 있는 수업은 건너뜀 (중복 생성 없음)
- 로그인 후 2초 내에도 자동으로 실행됨

**수업 카드 기능**
- 출석 체크: 예약자 이름 옆 `—` 버튼 클릭 → `✓` (출석) 토글
- 수업 취소: 수업을 "취소됨" 상태로 변경, 예약자 명단 보존
- 삭제: 수업과 관련 예약 모두 완전 삭제

**취소된 수업 특징**
- 빨간 배경으로 표시됨
- 예약자 명단이 그대로 보임 (누가 신청했었는지 확인 가능)
- 회원의 잔여 횟수는 차감되지 않음

### 탭 2: 회원 관리

**회원 등록 방법**
1. `+ 회원 등록` 버튼 클릭
2. 입력 항목:
   - **이름** (필수): 실명
   - **전화번호** (필수): 숫자만 입력 (`01012345678`)
   - 생년월일: 선택
   - **초기 비밀번호**: 기본값 `0000`
   - **수업 유형**: 그룹 / 프라이빗
   - **수강권 종류**: 4 / 8 / 12 / 기타
   - **잔여 그룹 횟수**: 실제 남은 수업 횟수 입력
   - **만료일**: 수강권 유효기간
3. **저장** 클릭

> 등록 후 회원에게 **전화번호 + 초기 비밀번호 `0000`** 을 알려주세요.

**회원 수정**
- 회원 카드의 `수정` 버튼 → 정보 변경 → 저장

**잔여 횟수 직접 조정**
- 회원 수정 → 잔여 그룹/프라이빗 횟수 직접 입력 → 저장

### 탭 3: 결제 요청

회원이 수강권 신청서를 제출하면 여기에 표시됩니다.

**처리 방법**
1. 실제 계좌로 입금 확인
2. `승인` 버튼 클릭 → 회원 수강권 자동 발급 (횟수 자동 증가)
3. 입금 확인 안 됐을 시 → `반려` 버튼

### 탭 4: 수업 등록

**개별 수업 등록**: 날짜, 시간, 유형, 정원 직접 입력

**토요일 일괄 생성**: 날짜 범위 지정 후 한 번에 생성
- 예) 2026-05-23 ~ 2026-08-30 범위 지정 시 그 사이 모든 토요일에 3타임 생성

### 탭 5: 백업/설정

**데이터 백업**
- `JSON 다운로드`: 모든 데이터 (회원 + 수업 + 예약 + 결제) 백업
- `CSV 다운로드`: 회원 목록만 엑셀에서 열 수 있는 형식으로 저장

**Firebase 연결 확인**
- `연결 상태 확인` 버튼으로 Firebase 쓰기 권한 테스트
- 오류 시 Firebase 규칙 설정 필요 (위 5번 섹션 참고)

## 7. 회원 대시보드 사용법

접속: `/dashboard.html` → 전화번호 + 비밀번호 입력

### 탭 1: 내 수강권

- 수강권 종류, 잔여 횟수, 사용 횟수 확인
- 진행 바로 남은 횟수 시각화
- 만료일 표시 (만료 시 ⚠️ 표시)
- 하단에 예약 내역 (예약완료 / 취소됨)

### 탭 2: 수업 예약

1. 달력에서 날짜 선택 (수업 있는 날에 파란 점 표시)
2. 수업 슬롯 확인 (잔여 인원, 진행 바)
3. `예약하기` 버튼 클릭
4. 예약 완료 → `✓ 예약완료` 배지 표시
5. 취소: `취소하기` 버튼 (전날 22:00까지만 가능)

**예약 불가 조건**
- 정원 마감 (16명)
- 전날 22:00 이후 취소 시도
- 잔여 수업 횟수 0
- 이미 예약된 수업

### 탭 3: 수강권 신청

1. 아래 계좌로 입금:
   - 카카오뱅크 `3333-29-4463177` (예금주: 김재휘)
2. 수업 유형 (그룹 / 프라이빗) 선택
3. 수강권 종류 선택
4. 메모 입력 (이체 시간 등)
5. `신청서 제출` 클릭
6. 관리자 승인 후 수강권 자동 발급

### 탭 4: 설정

- 내 정보 확인 (이름, 전화번호, 수업 유형 등)
- 비밀번호 변경 (현재 → 새 비밀번호 → 확인)

## 9. 데이터 업로드 및 공개 설정

### GitHub 저장소: Public vs Private

| 구분 | 특징 |
|------|------|
| **Public (권장)** | GitHub Pages 무료 사용 가능, 소스코드 공개 |
| **Private** | 소스코드 비공개, GitHub Pages는 유료 플랜($4/월) 필요 |

**권장: Public 저장소 사용**
- HTML/JS 코드가 공개되더라도 실제 데이터는 Firebase에 저장됨
- 비밀번호 등 민감한 정보: Firebase 규칙 또는 코드 내 위치 파악 필요

> 관리자 비밀번호(`s1102!`)가 `admin.html` 코드에 노출됩니다.  
> GitHub 저장소를 Public으로 설정할 경우 비밀번호 변경을 권장합니다.

### 업로드 대상 파일

GitHub에 올려야 하는 파일:

```
✅ 반드시 포함
├── index.html
├── dashboard.html
├── admin.html
├── .nojekyll
```

## 10. 데이터 백업 방법

### 방법 1: 관리자 패널에서 백업 (권장)

1. `/admin.html` 로그인
2. **백업/설정** 탭 선택
3. `JSON 다운로드` 버튼 → 전체 데이터 저장
4. `CSV 다운로드` 버튼 → 회원 목록 엑셀용 저장

**백업 파일명 예시:**
- `highend_backup_20260523.json`
- `highend_members_2026-05-23.csv`

### 방법 2: Firebase Console에서 직접 내보내기

1. https://console.firebase.google.com
2. 프로젝트 선택 → Realtime Database
3. 우측 상단 `⋮` (더보기) → **JSON 내보내기**

### 백업 권장 주기

| 상황 | 백업 주기 |
|------|-----------|
| 일반 운영 중 | 매월 1회 |
| 회원 급증 / 중요 업데이트 후 | 즉시 |
| 시즌 마무리 | 반드시 백업 |

백업 파일은 PC 및 Google Drive / iCloud 등에 보관 권장.

## 11. 유지 보수 가이드

### 정기 작업

**매주 (토요일 운영 전)**
- [ ] 관리자 패널 → 스케줄 탭에서 이번 주 수업 확인
- [ ] 예약 현황 확인 (몇 명 신청했는지)
- [ ] 결제 요청 탭 확인 → 미처리 요청 승인/반려

**매월**
- [ ] 관리자 → 백업/설정 → JSON 백업 다운로드
- [ ] 토요일 자동생성 버튼으로 다음 달 수업 생성
- [ ] 만료일 지난 회원 수강권 정리

**수시**
- [ ] 신규 회원 등록
- [ ] 수강권 횟수 조정 (결제 확인 후)
- [ ] 취소된 수업 처리

### 수업 생성 워크플로우

```
매월 초
  └─ 관리자 로그인
      └─ 스케줄 탭
          └─ "토요일 자동생성" 버튼 클릭
              └─ 이번 달 ~ 다음 달 4주치 수업 자동 생성됨
```

또는 수업 등록 탭 → 토요일 일괄 생성 → 날짜 범위 직접 지정

### 회원 수강권 갱신 워크플로우

```
회원 수강권 만료 또는 추가 구매 시
  └─ 회원: 대시보드 → 수강권 신청 탭 → 신청서 제출
      └─ 관리자: 결제 요청 탭 확인
          └─ 입금 확인 후 "승인" 클릭
              └─ 회원 잔여 횟수 자동 증가
```

또는 관리자가 직접: 회원 관리 탭 → 해당 회원 수정 → 잔여 횟수 조정

### 코드 수정 시 배포 절차

예: 수업 정원을 16명에서 20명으로 변경

1. VS Code로 `admin.html` 열기
2. `maxCapacity:16` → `maxCapacity:20` 변경
3. 저장
4. 터미널에서:
   ```bash
   cd "c:\Users\User\0. 강습 홈페이지"
   git add admin.html
   git commit -m "수업 정원 20명으로 변경"
   git push
   ```
5. 2분 후 자동 배포 완료

### 이미지 교체 방법

코치 사진이나 갤러리 사진을 새 사진으로 교체할 때:

1. 새 사진 파일을 `1. 사진 자료/` 폴더에 복사
2. `index.html`에서 이미지 경로 수정:
   ```html
   <!-- 기존 -->
   <img src="./1. 사진 자료/KakaoTalk_20260504_231115997.jpg">
   <!-- 수정 -->
   <img src="./1. 사진 자료/새파일명.jpg">
   ```
3. git add → commit → push

## 12. 문제 해결

### 회원 등록이 안 될 때

**원인**: Firebase 쓰기 권한 없음

**해결**:
1. admin.html → 백업/설정 탭 → `연결 상태 확인` 버튼
2. 빨간 오류 표시 시 Firebase Console에서 규칙 설정:
   ```json
   { "rules": { ".read": true, ".write": true } }
   ```

---

### 토요일 수업이 생성되지 않을 때

**확인 순서**:
1. 관리자 로그인 → 백업/설정 → 연결 상태 확인 (Firebase 권한 확인)
2. 스케줄 탭 → `토요일 자동생성` 버튼 수동 클릭
3. 결과 메시지 확인:
   - "✓ N개 수업 생성됨" → 정상
   - "이미 모두 존재합니다" → 이미 있음 (정상)
   - "생성 실패: ..." → Firebase 권한 문제

---

### 로그인이 안 될 때

**회원 로그인 실패**:
- 전화번호: 숫자만 입력 (`01012345678`)
- 초기 비밀번호: `0000`
- 비밀번호 분실 시 → 관리자가 회원 수정에서 비밀번호를 `0000`으로 초기화

**관리자 로그인 실패**:
- 아이디: `admin` (소문자)
- 비밀번호: `s1102!` (느낌표 포함)

---

### 홈페이지가 보이지 않을 때 (GitHub Pages)

1. GitHub 저장소 → Settings → Pages에서 배포 상태 확인
2. "Your site is live at..." 메시지 확인
3. 배포에 최대 5분 소요될 수 있음
4. 브라우저 캐시 삭제 후 재시도: `Ctrl + Shift + R`

---

### 수강권 잔여 횟수가 틀릴 때

관리자가 직접 수정 가능:

1. admin.html → 회원 관리 탭
2. 해당 회원 → `수정` 버튼
3. 잔여 그룹 횟수 또는 잔여 프라이빗 횟수 직접 수정
4. 저장

---

### 이미지가 안 보일 때 (GitHub Pages 배포 후)

이미지 파일이 git에 포함되지 않았을 경우:

```bash
cd "c:\Users\User\0. 강습 홈페이지"
git add "1. 사진 자료/"
git add "2. 커리큘럼 사진/"
git commit -m "이미지 파일 추가"
git push
```
